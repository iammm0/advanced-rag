# 分块路由器说明

## 功能概述

分块路由器（`ContentAnalyzer`）根据文档内容和元数据特征智能选择最合适的分块器。

## 路由策略（按优先级）

### 1. 递归分块器（高度结构化内容）

**适用场景：**
- 代码文件（.py, .js, .java等）
- 包含大量代码块的文档（>5个代码块）
- 包含大量LaTeX公式的学术论文（>10个公式）
- 包含大量结构化标记的文档（Markdown、HTML等）

**检测方法：**
- 检查文件扩展名（代码文件）
- 检查metadata中的`code_blocks`字段
- 检查文本中的代码块标记（```、缩进代码）
- 检查LaTeX公式模式（\begin{、\end{、$$等）
- 检查metadata中的`formulas`字段
- 检查结构化标记（#、<div、<table等）
- 检查metadata中的`tables`字段

### 2. 智能分块器（包含公式或表格）

**适用场景：**
- 包含数学公式的文档
- 包含表格的文档
- 需要保护公式和表格完整性的文档

**检测方法：**
- 检查metadata中的`formulas`字段
- 检查metadata中的`tables`字段
- 检查文本中的公式模式（$$、$、\[等）
- 检查文本中的表格标记（|、---等）

**特点：**
- 保护数学公式完整性
- 识别段落边界
- 识别列表和表格结构
- 根据内容类型调整分块大小

### 3. 语义分块器（需要保持语义连贯性）

**适用场景：**
- 长文档（报告、论文、文章）
- 包含大量段落的文档（>10个段落）
- 需要保持上下文连贯性的文档

**检测方法：**
- 检查文档长度（>1000字符）
- 检查段落数量（>10个段落）
- 检查句子数量（>50个句子）
- 检查平均段落长度（>200字符）
- 检查章节结构标记（第、章、节等）

**特点：**
- 基于语义相似度进行分块
- 保持语义连贯性
- 适合长文档和文章

### 4. 简单分块器（通用型）

**适用场景：**
- 短文档
- 简单文本
- 其他不满足上述条件的文档

**特点：**
- 按固定大小分割
- 使用分隔符优化分块位置
- 通用性强，适合大多数场景

## 使用示例

```python
from chunking.router import ContentAnalyzer

analyzer = ContentAnalyzer()

# metadata可能包含解析器提取的信息
metadata = {
    "formulas": [...],  # 公式列表
    "tables": [...],    # 表格列表
    "code_blocks": [...]  # 代码块列表
}

chunker_type, chunker = analyzer.route(text, metadata)

# chunker_type 可能是：
# - ContentAnalyzer.CHUNKER_TYPE_RECURSIVE
# - ContentAnalyzer.CHUNKER_TYPE_SMART
# - ContentAnalyzer.CHUNKER_TYPE_SEMANTIC
# - ContentAnalyzer.CHUNKER_TYPE_LEGACY

chunks = chunker.chunk(text, metadata)
```

## 元数据利用

分块路由器会充分利用解析器返回的metadata信息：

- **formulas**: 公式列表，用于检测包含公式的文档
- **tables**: 表格列表，用于检测包含表格的文档
- **code_blocks**: 代码块列表，用于检测包含代码的文档
- **file_type**: 文件类型，用于检测代码文件

这些信息使路由决策更加准确。

## 日志输出

路由器会输出详细的日志信息，帮助理解路由决策：

```
✓ 检测到代码文件，使用递归分块器
✓ 检测到 5 个代码块，使用递归分块器
✓ 检测到 10 个公式，使用智能分块器
✓ 检测到 3 个表格，使用智能分块器
✓ 检测到长文档 (5000 字符, 15 段落)，使用语义分块器
使用简单分块器（通用型）
```

## 错误处理

- 如果语义分块器初始化失败，会自动回退到其他分块器
- 如果检测过程中出错，会记录警告并继续处理
- 空文本会使用简单分块器

## 最佳实践

1. **传递metadata**: 尽量传递解析器返回的metadata，以获得更准确的路由决策
2. **监控日志**: 查看路由决策日志，了解文档被路由到哪个分块器
3. **性能考虑**: 语义分块器需要计算嵌入向量，可能较慢，适合长文档
