# 高阶RAG系统全流程核心知识点（按环节拆解，无图谱，纯干货）

基于已掌握的基础RAG流程，融合**重排模型、GraphRAG、自动化评测**三大高阶模块，按「数据处理→索引构建→检索召回→生成输出→评测优化」全流程梳理，每个环节标注**核心动作、高阶升级点、技术复用、关键要点**。

## 一、数据层：文档处理（基础RAG底座，高阶无新增环节，仅优化策略）

### 核心流程

文档采集→文档解析→智能分块→元信息绑定

### 基础核心点

1. 采集：多源获取（本地/爬虫/业务库），兼容PDF/Word/图片/代码等格式；

2. 解析：PaddleOCR处理图文，LangChain解析器提取纯文本，解决表格/代码块错乱问题；

3. 分块：核心是「粒度适中+语义完整」，基础用固定token长度（如512）拆分；

4. 元信息：为每个分块打「来源文档/业务标签/课程/行业」标签，支撑后续过滤检索。

### 高阶优化点（需重点掌握）

- 分块策略升级为**规则分块+语义分块混合**：规则分块保证代码/公式/表格完整性，语义分块（基于大模型/Ollama）避免完整语义被拆分；

- 分块后做**去重预处理**：剔除重复文本块，减少后续索引冗余。

## 二、索引层：双索引构建（基础RAG→高阶核心升级，新增**知识图谱索引**，与原向量索引形成双索引）

### 2.1 向量索引（基础RAG原有，无核心变化，复用现有技术）

1. 向量化：分块文本输入Embedding模型（如BGE-small-zh），生成固定维度向量（如768维），做归一化处理；

2. 入库优化：Qdrant/FAISS创建集合，绑定分块元信息，构建HNSW索引（平衡检索速度与精准度）；

3. 核心要点：向量维度与模型匹配，元信息必须和向量一一对应，支撑后续混合检索过滤。

### 2.2 知识图谱索引（GraphRAG核心，新增环节，复用简历Neo4j）

1. 知识抽取：分块文本→大模型/Ollama（定制Prompt）→抽取**实体-关系-属性**三元组

    - 实体：专业术语/名词（如LangChain、SQL注入、COMSOL），需做归一化（如统一大小写/简称）；

    - 关系：实体间明确关联（如“基于XX开发”“用于XX场景”“包含XX漏洞”，拒绝模糊的“相关”）；

    - 属性：实体特征（如LangChain：Python框架、Qdrant：向量数据库）。

2. 图谱构建：三元组清洗→Neo4j创建节点/边→绑定属性→建立节点索引

    - 节点：对应实体，标注实体类型（如“框架”“数据库”“漏洞”）；

    - 边：对应关系，标注关系类型（如“基于”“应用于”）；

    - 核心：避免重复节点、错误关联边，节点建索引提升检索速度。

3. 图谱优化：实体去重+关系归一化+图谱剪枝，剔除无关关系/冗余节点，保证图谱简洁。

## 三、检索层：高阶混合检索（已做基础混合检索，核心升级**重排模型+图谱检索融合**）

### 核心流程

用户查询输入→查询预处理→多维度召回→重排模型二次排序→结果筛选

### 3.1 查询预处理

1. 纯文本化：清洗查询中的无效字符，提取核心关键词/实体；

2. 双处理：① 查询向量化（用于向量检索）；② 抽取查询实体（用于图谱检索）。

### 3.2 多维度召回（基础混合检索升级，融合**文本检索+图谱检索**，并行执行）

1. 文本混合检索（已实现，复用）：向量检索（余弦/欧氏距离）+关键词检索（BM25）+过滤检索（按元信息标签），召回Top50左右文本分块；

2. 图谱检索（新增，GraphRAG核心）：基于查询抽取的实体→Neo4j检索→召回**实体本身+关联节点+关系**，转化为文本化知识；

3. 结果融合：将文本分块与图谱文本化知识合并，形成「初步召回结果集」，为后续重排做素材。

### 3.3 重排模型二次排序（高阶核心升级1，解决召回结果排序乱、相关性低）

1. 核心逻辑：轻量级重排模型（如BGE-reranker-base）对初步召回结果集，按「查询与内容的真实关联度」打分排序；

2. 关键操作：

    - 模型选型：选轻量级开源模型，避免推理耗时过高；

    - 结果截断：重排后取Top10-Top20高关联度结果，剔除低关联内容；

    - 性能优化：对重排模型做量化（如INT8），将推理耗时控制在200ms内；

3. 核心要点：重排是**对召回结果的二次筛选**，而非重新检索，核心目标是提升“优质内容在前”的概率。

## 四、生成层：知识融合输出（基础RAG原有，新增**文本+图谱知识融合**，衔接AI Agent）

1. 知识拼接：将重排后的文本分块+图谱检索的关联知识，按关联度高低拼接，形成**检索知识上下文**；

2. 大模型生成：将「用户查询+检索知识上下文」传入大模型/Ollama，按定制Prompt生成答案；

3. 高阶要点：Prompt中明确要求大模型**基于检索知识回答**，禁止编造，同时融合图谱中的关联关系，让答案更完整；

4. 衔接Agent：将整个检索生成流程封装为**LangChain Tool**，让AI Agent可直接调用，为ReAct/Plan-and-Execute范式提供知识支撑。

## 五、评测层：自动化评测流水线（高阶核心升级2，企业级落地必备）

### 核心目标

量化RAG效果，实现“一键评测+问题告警+迭代优化”，替代人工测试

### 核心模块&知识点

1. 评测数据集构建：针对业务场景（课程答疑/安全测试/行业分析），构建**100+条「问题-标准答案」样本集**，答案需精准、唯一；

2. 核心量化指标（5个核心，全覆盖）：

    - 检索指标：召回率（是否找得到相关知识）、精确率（是否找错无关知识）；

    - 生成指标：答案相关性（答案与问题的贴合度）、幻觉率（答案中编造内容的比例）；

    - 工程指标：响应时间（全流程耗时，需控制在用户可接受范围）；

3. 自动化执行：Python+Pytest+LangChain-Eval编写脚本，一键跑通「查询→检索→生成→指标计算」全流程；

4. 告警与报告：基于Prometheus做基础监控，对**幻觉率＞10%、响应时间＞500ms、召回率＜80%** 做日志告警；生成可视化评测报告，明确优化点；

5. 迭代优化：基于评测报告，针对性优化前序环节（如召回率低优化检索策略，幻觉率高优化重排/知识拼接）。

## 六、高阶RAG核心融合点（贯穿全流程，必掌握）

1. 双索引协同：**向量索引管“语义相似”，知识图谱索引管“知识关联”**，二者缺一不可，共同支撑检索；

2. 重排模型的定位：仅作用于「初步召回结果集」，不参与检索环节，核心是“提优”而非“补漏”；

3. 与AI Agent的衔接：整个高阶RAG封装为标准化Tool，遵循LangChain Tool规范，让Base Agent/子智能体可直接调用，为ReAct思考环节提供知识；

4. 性能与精准度平衡：所有高阶操作（重排、图谱检索）都需做性能优化（量化、索引、截断），避免因高阶升级导致全流程响应过慢，影响Agent使用体验。

## 七、全流程关键避坑点（高阶升级易踩，直接记）

1. 知识抽取：大模型幻觉导致三元组错误，需定制**精准抽取Prompt**，并做人工校验抽样；

2. 混合检索：各检索策略权重失衡，需基于业务场景调试权重（如技术问答偏向量检索，精准查询偏关键词检索）；

3. 重排模型：推理耗时过高，优先选轻量级模型+模型量化，而非大模型；

4. 图谱检索：实体抽取错误导致召回无关知识，需优化查询实体抽取的Prompt；

5. 自动化评测：标准答案标注模糊，导致指标计算失真，样本集答案需做到“唯一、精准、无歧义”。
